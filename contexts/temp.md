Mobile App onboarding:

1. User opens the mobile app
2. A page appears showing onbarding slides
3. User press on continue and google sign in starts happening, the button starts loading with circular indicator
4. in the backend, it checks for existing employee of any
5. if employee of any branch, then it directly gets the user into home page
6. if not then the user is shown the branches to join/create
7. if join, then the employee is created with REQUESTED status
8. if created, then the employee account is created with OWNER role pre-made for that branch with permissions "ALL"
9. and then the user is directed to home page
10. There will be a parent layout having bottom nav (center docked floating button) bar as primary and the upper screen will be changing based on the selected screen
11. The navbar will contain sections like billing, orders, employees, procurment, finance, etc and the best thing is the order can be managed locally
